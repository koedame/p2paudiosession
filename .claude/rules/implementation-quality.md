# 実装品質ルール

## 絶対禁止

1. **形骸化実装禁止**
   - 空の関数で「実装完了」としない
   - `todo!()` / `unimplemented!()` を残したままコミットしない
   - ログ出力だけで実際の処理がない実装は禁止

2. **仕様違反禁止**
   - `docs-spec/` の仕様に反する実装をしない
   - 仕様変更が必要な場合は先にADRを作成する

3. **品質チェックスキップ禁止**
   - `cargo fmt` / `cargo clippy` / `cargo test` をスキップしない
   - 警告を無視してコミットしない

## 推奨事項

- エラーハンドリングは適切に行う
- パニックは本当に回復不能な場合のみ
- コメントは「なぜ」を説明する（「何を」はコードを読めばわかる）

## このプロジェクト固有のルール

- **低遅延優先**: パフォーマンスに影響する変更は慎重に
- **音質優先**: 音声処理の品質を落とさない
- 詳細は [ADR-008](../../../docs-spec/adr/ADR-008-zero-latency-mode.md) を参照
