# テスト品質ルール

## 絶対禁止

1. **テストの削除・無効化禁止**
   - 既存テストを削除してはならない
   - `#[ignore]` で無効化してはならない
   - テストが失敗する場合は、テストではなく実装を修正する

2. **テストの形骸化禁止**
   - `assert!(true)` のような意味のないアサーション禁止
   - テスト対象の動作を実際に検証すること

3. **テストカバレッジの低下禁止**
   - 新機能追加時は対応するテストも追加
   - 公開 API には必ずテストを書く

## 推奨事項

- テストは独立して実行可能であること
- テスト名は動作を説明する名前にする
- エッジケースもテストする

## 例外

- 明らかに不要になったテスト（削除された機能のテスト）は削除可能
- ただし、削除理由をコミットメッセージに明記すること
