# トラブルシューティング

jamjamで問題が発生した場合の解決方法をまとめています。

## 接続の問題

### サーバーに接続できない

**症状**: 「サーバーに接続できません」エラーが表示される

**原因と解決策**:
1. **インターネット接続を確認**
   - 他のウェブサイトにアクセスできるか確認
   - Wi-Fiまたは有線接続が安定しているか確認

2. **シグナリングサーバーURLを確認**
   - 設定画面でサーバーURLが正しいか確認
   - デフォルト: `wss://signaling.example.com`（テストサーバー情報は管理者に確認）

3. **ファイアウォール設定**
   - jamjamがネットワークアクセスを許可されているか確認
   - Windows: 「Windows セキュリティ」→「ファイアウォールとネットワーク保護」
   - macOS: 「システム設定」→「ネットワーク」→「ファイアウォール」

### ルームが見つからない

**症状**: 「Room not found」エラーが表示される

**解決策**:
1. 招待コードが正しく入力されているか確認（6文字の英数字、大文字）
2. ルームがまだ存在しているか確認（ホストが退出するとルームは削除される）
3. コードを再度ホストに確認する

### パスワードエラー

**症状**: 「Invalid password」エラーが表示される

**解決策**:
1. パスワードが正しいか確認
2. 大文字・小文字を区別するため、正確に入力
3. ホストにパスワードを再確認する

### 接続がタイムアウトする

**症状**: 接続が完了せず、タイムアウトする

**原因と解決策**:
1. **NATの問題**
   - 対称NAT（Symmetric NAT）環境では直接接続が難しい場合がある
   - VPNを使用している場合は無効にして試す

2. **ネットワーク遅延**
   - 安定した回線を使用する
   - 可能であれば有線接続を使用

## 音声の問題

### 音声が聞こえない

**症状**: 接続は成功したが、相手の音声が聞こえない

**解決策**:
1. **出力デバイスを確認**
   - 設定画面で正しい出力デバイスが選択されているか確認
   - システムの音量がミュートになっていないか確認

2. **入力デバイスを確認（相手側）**
   - 相手のマイクが正しく設定されているか確認
   - 入力レベルメーターが反応しているか確認

3. **ミュート状態を確認**
   - マイクミュートボタンがオンになっていないか確認

### 音声が途切れる・ノイズが入る

**症状**: 音声が断続的に途切れる、またはノイズが混入する

**解決策**:
1. **プリセットを調整**
   - 「balanced」または「high-quality」プリセットに変更
   - ネットワーク状態が悪い場合、より保守的な設定を使用

2. **バッファサイズを増やす**
   - 設定画面でオーディオバッファサイズを増やす
   - 128 → 256 または 256 → 512 に調整

3. **ネットワーク帯域を確認**
   - 他の帯域を消費するアプリ（動画ストリーミング等）を停止
   - 可能であれば有線接続を使用

### 遅延が大きい

**症状**: 音声の遅延が気になる

**解決策**:
1. **プリセットを変更**
   - 「zero-latency」または「ultra-low-latency」を試す
   - 推奨設定が表示されている場合はそれに従う

2. **バッファサイズを減らす**
   - 設定画面でオーディオバッファサイズを減らす
   - 注意: 小さすぎると音声が途切れる場合がある

3. **ネットワーク品質を確認**
   - RTT（往復遅延時間）が低いほど低遅延になる
   - 日本国内光回線で10-25ms程度が目安

## アプリケーションの問題

### アプリが起動しない

**Windows**:
1. Visual C++ 再頒布可能パッケージがインストールされているか確認
2. .NET Framework が最新か確認
3. 管理者として実行を試す

**macOS**:
1. 「システム設定」→「プライバシーとセキュリティ」で許可
2. 「このまま開く」を選択

**Linux**:
1. 必要なライブラリがインストールされているか確認:
   ```bash
   sudo apt-get install libasound2 libssl3 libwebkit2gtk-4.1-0
   ```
2. AppImageの場合、実行権限を付与:
   ```bash
   chmod +x jamjam.AppImage
   ```

### 設定が保存されない

**症状**: アプリを再起動すると設定がリセットされる

**解決策**:
1. 設定ファイルの場所を確認:
   - Linux: `~/.config/jamjam/config.toml`
   - macOS: `~/Library/Application Support/jamjam/config.toml`
   - Windows: `%APPDATA%\jamjam\config.toml`

2. ディレクトリの書き込み権限を確認

### クラッシュする

**症状**: 使用中にアプリが突然終了する

**解決策**:
1. アプリを最新版に更新
2. 設定ファイルを削除して初期化:
   ```bash
   # Linux/macOS
   rm -rf ~/.config/jamjam/

   # Windows (PowerShell)
   Remove-Item -Recurse $env:APPDATA\jamjam
   ```

## テストユーザー向け追加情報

### ログの取得

問題報告時にはログが役立ちます:

1. アプリを起動
2. 問題を再現
3. ログファイルを取得:
   - Linux: `~/.config/jamjam/logs/`
   - macOS: `~/Library/Logs/jamjam/`
   - Windows: `%APPDATA%\jamjam\logs\`

### 問題報告

問題を報告する際は以下の情報を含めてください:

1. **環境情報**
   - OS（Windows/macOS/Linux）とバージョン
   - jamjamのバージョン
   - 使用しているオーディオデバイス

2. **問題の詳細**
   - 何をしたか（手順）
   - 期待した動作
   - 実際の動作
   - エラーメッセージ（ある場合）

3. **ログファイル**（可能であれば）
